## 二、实验设计（35 个场景）

```
★ 所有场景均包含「历史功率时序」作为输入特征

层1（29 个场景）── 功率+风速结构 vs 去SCADA

  P 组（功率自回归基准）：
    P0  : 仅历史功率（下界基准）

  M 组（含SCADA，15个）：
    M0  : SCADA机舱风速 + 历史功率
    M1  : LiDAR单距离（40~300m，10个）+ SCADA + 历史功率
    M2  : 近距组（40~120m）+ SCADA + 历史功率
    M3  : 中距组（150~210m）+ SCADA + 历史功率
    M4  : 远距组（240~300m）+ SCADA + 历史功率
    M5  : 全距离 LiDAR + SCADA + 历史功率

  N 组（无SCADA，13个）：
    N1  : LiDAR单距离（40~300m，10个）+ 历史功率
    N2  : 近距组 + 历史功率
    N3  : 中距组 + 历史功率
    N4  : 远距组 + 历史功率
    注：N5（全距无SCADA）= L3a

层2（4 个场景）── 气象特征增益（均含历史功率）
层3（2 个场景）── 去SCADA验证（均含历史功率）
```

**三模型配置**：
- **LightGBM / XGBoost**：所有 35 个场景（直接多步策略：每步长独立建模）
- **LSTM**：15 个组合对比场景（P0/M0/M2~M5/N2~N4/E1~E4/L3a/L3b，不含逐距离扫描）

## 四、关键结论（v5 修订）

### 结论1：所有场景均含历史功率（验证正确性）
所有 35 个场景的 `feat_cols` 均以 `["power"]` 作为最后一个元素，历史功率时序（LOOK_BACK=6步，60min）始终作为输入特征。这已在代码中明确，并反映在所有场景显示名称中（含"+历史功率"）。

### 结论2：LSTM 在长步长（+30/+60min）上全面优势显著
| 步长 | LSTM vs LightGBM 平均优势（11场景） |
|-----|--------------------------------|
| +10min | 约 +2kW（LSTM有时更差） |
| +20min | 约 −18kW（LSTM更好） |
| +30min | 约 −50kW（LSTM明显更好）|
| +60min | 约 **−77kW**（LSTM大幅更好）|

在 +60min 步长，LSTM 比 LightGBM 平均好约 **77kW**（约 8~9% 的相对改善）。

### 结论3：树模型在 +10min 中，P0 仍最优（反直觉）
- LightGBM/XGBoost：P0（仅历史功率）+10min=432kW，优于所有含风速场景
- LSTM：P0+10min=417kW，仍然最优
- 解释：60min历史功率序列已充分编码近期风速信息（P∝V³），额外风速特征在此窗口下提供的增量有限

### 结论4：SCADA对近距LiDAR有轻微帮助，对中远距有干扰（三模型一致）
- M2（近距+SCADA）vs N2（近距）：LSTM 443 vs 467（+10min），SCADA有益 Δ=+24kW
- M5（全距+SCADA）vs L3a（全距）：LSTM 461 vs 446（+10min），去SCADA反而更好 Δ=−15kW

### 结论5：L3b 异常（LSTM+气象特征恶化）
L3b（全距LiDAR+气象+历史功率，无SCADA）在LSTM下+10min=511kW，比L3a（446kW）**差65kW**。LiDAR + 全气象（30列）+历史功率 的高维特征组合在LSTM框架下发生过拟合/噪声放大，需谨慎。

## 五、三模型 LSTM/LightGBM/XGBoost 定位总结

| 模型 | 特点 | 推荐场景 |
|-----|------|---------|
| **LSTM** | 短步长持平/略差，长步长（+30/+60min）**大幅优势** | 工程部署（预测30~60min）|
| **LightGBM** | 快速、稳定，短步长最好的树模型 | 快速实验、特征重要性分析 |
| **XGBoost** | 与LightGBM接近，长步长略优 | 集成模型备选 |
